Qwen Text LoRA Finetune Report

Small controlled LoRA experiment on 100 synthetic text-poster images. The dataset includes English-only, Arabic-only, stacked bilingual, and structured bilingual layouts.

Training Setup

Model: Qwen-Image-Edit-2511 used as text-to-image through the shared Qwen Image transformer. Training resolution: 512x512. Adapter: rank 8 attention LoRA on Q/K/V/output projections. Trainable parameters: 14,745,600. Steps: 800. Learning rate: 5e-05.

Saved adapter: pytorch_lora_weights.safetensors

Result Summary

This run does not solve Arabic text rendering. The LoRA learned the synthetic poster prior strongly and improves simple English poster text, but the before/after generations do not show reliable exact Arabic or bilingual transcription. Treat it as a useful pipeline proof, not a quality fix.

The main evidence is visual: compare the base and LoRA columns below. The LoRA changes layout/style toward the synthetic dataset, makes English prompts cleaner in this small test, and still leaves Arabic glyph identity/word correctness weak.

Recommendation

For a real Arabic poster fix, this result says the pipeline is working but the dataset is too small and too synthetic. Next run should use thousands of high-quality rendered poster crops, include exact OCR/CLIP-style text scoring, and likely target the non-catastrophic text-sensitive layer bands from the ablation probe rather than only generic attention LoRA.
